# RUMS Backend — Java Spring Boot API Specification

## 1. System Overview

**RUMS (Regulatory Utility Management System)** is the internal back-office MIS for the
Rwanda Utilities Regulatory Authority (RURA). The backend is a RESTful JSON API consumed
exclusively by the Next.js 14 frontend described in this repository.

### Scope of the backend
- Authentication & authorisation (JWT, role-based)
- Full CRUD for all 10 modules
- Automatic audit logging on every mutating action
- Automatic alert generation on configurable triggers
- Dashboard aggregate endpoints
- File upload/download for CLMS documents and Report files
- Scheduled jobs: licence expiry alerts, SLA breach detection

## 3. Architecture

Root package: `rw.rura.rums`. Top-level sub-packages: `config`, `security`, `exception`, `audit`, `alert`, `module`, plus `RumsApplication.java`.
Each module lives under `module/{name}/` and contains sub-packages: `entity`, `repository`, `service`, `controller`, `dto`.
Modules: `auth`, `dashboard`, `licenses`, `complaints`, `compliance`, `fraud`, `clms`, `workflows`, `alerts`, `audit`, `reports`, `users`.

## 4. Database

**Database name:** `rums_db`
**Schema:** `public`
**Charset:** `UTF-8`

All tables use `UUID` primary keys generated by the database (`gen_random_uuid()`).
All timestamps are stored as `TIMESTAMPTZ` (UTC).
Soft-delete is used on `users` only (via `deleted_at` column).
All other deletes are hard deletes (no records are deleted in normal operation).

  title               VARCHAR(300)   NOT NULL,

## 5. Security & Authentication

### JWT Configuration
- Algorithm: **HS256**
- Access token expiry: **8 hours**
- Refresh token expiry: **7 days**
- Secret: externalized to `application.properties` (`rums.jwt.secret`)
- Token delivered in JSON response body; frontend stores in memory / `httpOnly` cookie

### Endpoints that do NOT require authentication
```
POST /api/auth/login
POST /api/auth/refresh
GET  /swagger-ui.html
GET  /v3/api-docs/**
```

All other `/api/**` endpoints require a valid JWT Bearer token.

### Role-based access matrix

| Role | Read | Create | Update status | Full edit | User mgmt | Delete |
|---|---|---|---|---|---|---|
| `admin` | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `supervisor` | ✓ | ✓ | ✓ | ✓ | — | — |
| `auditor` | ✓ | Compliance / Audit only | ✓ | — | — | — |
| `analyst` | ✓ | ✓ | ✓ | — | — | — |
| `viewer` | ✓ | — | — | — | — | — |

Enforce with `@PreAuthorize` on service methods.

### Password rules
- Minimum 10 characters, at least 1 uppercase, 1 digit, 1 special character
- Hashed with **BCrypt** (strength 12)
- Invite flow: system generates a temporary password, emails it, user must change on first login

### CORS
Allow origin `http://localhost:3000` (and the production frontend domain via config).
Allow methods: `GET, POST, PUT, PATCH, DELETE, OPTIONS`.
Allow headers: `Authorization, Content-Type`.

## 6. Standard API Conventions

### Base URL
```
/api
```

### Response envelope
All responses use a consistent wrapper:

```json
{
  "success": true,
  "data": { ... },
  "message": "Optional human-readable message",
  "timestamp": "2026-03-22T08:14:00Z"
}
```

Errors:
```json
{
  "success": false,
  "error": "RESOURCE_NOT_FOUND",
  "message": "License with ID abc123 not found",
  "timestamp": "2026-03-22T08:14:00Z"
}
```

### Pagination
All list endpoints accept:
```
?page=0&size=20&sort=createdAt,desc
```

Paginated responses return:
```json
{
  "success": true,
  "data": {
    "content": [...],
    "totalElements": 150,
    "totalPages": 8,
    "number": 0,
    "size": 20
  }
}
```

### Filtering
All list endpoints accept filter query parameters matching entity fields.
Example: `GET /api/licenses?status=active&sector=Telecom`

### Date format
ISO-8601: `yyyy-MM-dd` for dates, `yyyy-MM-dd'T'HH:mm:ss'Z'` for timestamps.

### IDs
All IDs are UUIDs returned as strings.

### 7.1 Authentication — `/api/auth`

#### `POST /api/auth/login`
**Body:**
```json
{ "email": "k.mutabazi@rura.rw", "password": "secret" }
```
**Response:**
```json
{
  "accessToken": "eyJ...",
  "refreshToken": "eyJ...",
  "expiresIn": 28800,
  "user": { "id": "...", "name": "Kagiso Mutabazi", "role": "admin", "department": "ICT" }

```
Sets `updated_at = last_login` on the user record.
Records an `AuditLogEntry` with `action = login`, `module = System`.

#### `POST /api/auth/refresh`
**Body:** `{ "refreshToken": "eyJ..." }`
**Response:** New `accessToken` and `refreshToken`.

#### `POST /api/auth/logout`
**Auth required.** Invalidates refresh token (store revoked token JTI in an in-memory set or `revoked_tokens` DB table). Records `action = logout`.

#### `PATCH /api/auth/change-password`
**Auth required.**
**Body:** `{ "currentPassword": "...", "newPassword": "..." }`
Records `action = password_reset`.

Query params: `status`, `sector`, `province`, `search` (searches `operator_name`, `license_number`).

**Body:** `LicenseCreateRequest`

{
  "licenseNumber": "RURA-TLC-2026-031",

  "contactPerson": "...",

Records audit: `action = create`.

#### `PUT /api/licenses/{id}`
**Roles:** admin, supervisor.
**Body:** same as create (all fields optional except those required).
Records audit: `action = update`, `changes = {field: {before, after}}` for changed fields.

**Body:** `{ "status": "active" | "suspended" | "revoked" | "expired" }`
**Roles:**
- `active` → admin, supervisor (approval)
- `suspended` → admin, supervisor
- `revoked` → admin only

Triggers alert creation for `revoked` or `suspended`.

#### `GET /api/licenses/trend`
Returns `LicenseTrendPoint[]` for the last 6 months (same logic as dashboard).

## 8. Audit Service

`AuditService` is a Spring `@Service` injected into every other service. It must be called on
every mutating operation.

```java
public void log(
    User actor,           // currently authenticated user
    AuditAction action,
    AuditModule module,
    String entityId,
    String entityLabel,
    HttpServletRequest request,                       // for IP + user-agent
    Map<String, ChangeDto> changes                    // nullable
)
```

`ChangeDto` is `{ String before; String after; }`.

The service persists an `AuditLog` entity to the `audit_log` table immediately (synchronously, within the same transaction as the business operation).

---

## 9. Alert Service

`AlertService` creates `Alert` records. Called programmatically by other services, and by scheduled jobs.

```java
public void createAlert(
    AlertType type,
    String title,
    String message,
    AlertSeverity severity,
    AuditModule relatedModule,
    String relatedEntityId   // nullable
)
```

Prevents duplicate alerts: before inserting, check if an `unread` alert of the same `type` + `relatedEntityId` already exists. If so, skip creation.

## 10. Scheduled Jobs (`AlertScheduler`)

Run on server startup and then on cron schedule.

### Licence Expiry Alerts
**Schedule:** `@Scheduled(cron = "0 0 6 * * *")` — 06:00 UTC daily

Query `licenses` where `status = active` and `expires_at <= NOW() + 30 days`.
For each result:
- If `expires_at <= NOW()` → set `status = expired`, create alert `license_expiry` / critical
- If `expires_at <= NOW() + 9 days` → create alert `license_expiry` / critical
- If `expires_at <= NOW() + 30 days` → create alert `license_expiry` / warning

### SLA Breach Detection
**Schedule:** `@Scheduled(cron = "0 0 8 * * *")` — 08:00 UTC daily

SLA threshold: 14 days. Query `complaints` where `status IN (open, under_review)` and `filed_at < NOW() - 14 days`.
For each: create alert `complaint_sla_breach` / warning.

### Workflow Stall Detection
**Schedule:** `@Scheduled(cron = "0 30 8 * * *")` — 08:30 UTC daily

Query `workflows` where `status IN (active, paused)` and `started_at < NOW() - 5 days` and no step completed in last 5 days.
For each: create alert `workflow_stalled` / warning.

### Fraud Loss Threshold
**Schedule:** `@Scheduled(cron = "0 0 9 * * MON")` — Monday 09:00 UTC weekly

Sum `estimated_loss_rwf` of `fraud_cases` where `status NOT IN (dismissed)` in current quarter.
If sum > 200,000,000 → create alert `threshold_exceeded` / critical.

## 12. Sequence Auto-Generation

The `caseNumber` / `referenceNumber` / `licenseNumber` fields are auto-generated in the service layer using the format shown below. Use a DB sequence or a `SELECT COUNT(*)+1` query with a unique constraint.

| Entity | Format |
|---|---|
| Complaint | `RURA-CMP-{YYYY}-{NNNN}` (4-digit zero-padded) |
| Fraud Case | `RURA-FRD-{YYYY}-{NNNN}` |
| CLMS Case | `CLMS-{YYYY}-{NNNN}` |

For Licenses, the `licenseNumber` is provided by the user (it encodes sector and category).

## 13. File Storage

- Upload root: configurable via `rums.files.upload-dir` in `application.properties` (default: `./uploads/`)
- Subdirectories: `clms-documents/` and `reports/`
- Stored filename: `{UUID}_{sanitised-original-filename}` (strip special chars)

- Download endpoint streams the file with `Content-Disposition: attachment; filename="..."` header

## 16. Error Codes Reference

| Code | HTTP Status | Meaning |
|---|---|---|
| `RESOURCE_NOT_FOUND` | 404 | Entity with given ID does not exist |
| `DUPLICATE_KEY` | 409 | Unique constraint violated (e.g. licence number already exists) |
| `VALIDATION_ERROR` | 400 | Bean validation failed; `data` contains field errors |
| `FORBIDDEN` | 403 | Authenticated but insufficient role |
| `UNAUTHORIZED` | 401 | No/invalid JWT |
| `INVALID_STATUS_TRANSITION` | 422 | e.g. trying to `approve` an already `revoked` licence |

| `UNSUPPORTED_FILE_TYPE` | 415 | File extension not in allowed list |
| `INTERNAL_ERROR` | 500 | Unexpected server error |
